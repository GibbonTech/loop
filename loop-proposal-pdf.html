Loop — Proposition Technique

Vision

Vous faites du portage salarial pour chauffeurs VTC. Vous gérez déjà tout le back-office métier (contrats, paie, fiscalité).

Le besoin aujourd'hui est l'interface client. Un portail où vos chauffeurs voient leurs revenus, téléchargent leurs documents et soumettent leurs frais, sans intervention manuelle de votre part.

Le Périmètre

Un portail web et une application mobile native.

  • Dashboard Revenus par jour/mois, détail par plateforme
  • Répartition CA → indemnités km → brut → net
  • Sync Plateformes Uber, Bolt, Heetch centralisés automatiquement
  • Documents Bulletins de paie (PDF), contrats, attestations
  • Notes de Frais Upload factures, tracking TVA
  • Profil Infos chauffeur + véhicule (modifiable)

Approche Technique

Une seule base de code pour deux interfaces. Mise à jour en temps réel via connexion API aux plateformes VTC.

Points de Clarification

Pour garantir une implémentation précise, nous devons valider ces workflows.

Calcul Salaire Net

Le calcul CA → Net est-il fait par votre fiscaliste avant affichage, ou l'app doit-elle intégrer vos règles de calcul ?

Source des Données

Privilégie-t-on la sync auto (API) ou l'import CSV par le chauffeur ? Quel est le fallback ?

Origine Bulletins de Paie

Proviennent-ils d'un logiciel tiers (Silae, PayFit) via API, ou d'un upload manuel de votre part ?

Validation Notes de Frais

Simple tracking pour le chauffeur, ou workflow de validation par vos soins avant remboursement ?

Périmètre Plateformes

Uber, Bolt, Heetch sont confirmés. Y a-t-il d'autres acteurs majeurs (FreeNow, LeCab) pour le MVP ?

Gestion Documents

Outre contrat/mutuelle, y a-t-il des documents spécifiques nécessitant une signature électronique ?

Modification Profil

Le chauffeur peut-il modifier son SIRET/Véhicule librement, ou cela requiert-il votre validation ?

Timeline & Priorité

L'usage sera-t-il majoritairement mobile (app) ou desktop ? Quelle est la date cible de lancement ?
